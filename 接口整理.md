### 成长伙伴_后端API功能清单v1.0
#### 说明：所有接口统一前缀`/api/v1`，未标注「公开」的接口均需携带JWT Token，部分接口需额外校验角色权限；`WS`为WebSocket接口，非HTTP接口。

### 一、认证模块（Auth）
| 请求方法 | 接口路径 | 权限要求 | 功能说明 |
| ---- | ---- | ---- | ---- |
| POST | /auth/login | 公开 | 统一登录入口，返回access_token + refresh_token + 用户信息 + 角色 |
| POST | /auth/refresh | 公开 | 用refresh_token换新access_token，刷新登录态 |
| POST | /auth/logout | JWT | 注销登录，refresh_token加入Redis黑名单 |
| GET | /auth/me | JWT | 获取当前登录用户基础信息（角色/班级/用户ID） |
| PATCH | /auth/password | JWT | 修改密码（验证旧密码），修改后所有Token失效 |

## 📋 后端功能清单总览（72条路由，10大模块）

### 模块分布

| 模块 | 路由数 | 核心职责 |
|------|--------|---------|
| 认证（Auth） | 5 | 登录/刷新/注销/改密 |
| 管理员（Admin） | 24 | 学校/班级/用户/权限分配/升年级 |
| 老师端（Teacher） | 22 | 打分/广播/盲盒/题库/报告 |
| 学生端（Student） | 12 | 伙伴/行为查看/对战/年历 |
| 家长端（Parent） | 9 | 只读查看孩子数据 |
| 对战（Battle） | 6+1WS | 房间管理+实时对战 |
| 广播（WebSocket） | 1WS | 全局实时推送通道 |
| 公开接口 | 4 | 模板/健康检查/全局配置 |
| 阳光章（二期预留） | 9 | 盖章/评选/七色配置 |

### 关键约束（重点审查）

**① 打分接口是最核心的**，必须在单个数据库事务内完成：写 `behavior_records` → 更新 `partners.growth_points` → 写 `growth_records` → 检测进化 → 检测满级，任何一步失败全部回滚。

**② 老师每次操作班级前**，必须查 `teacher_class_assignments` 校验权限，这是防越权的关键，不能省略。

**③ WebSocket 推送必须是异步 goroutine**，绝对不能阻塞 HTTP 响应返回。

**④ 成长值只增不减**，所有写成长值的地方都要检查 delta 必须为正整数。

**⑤ 敏感字段永远不出响应体**，`real_name_enc`、`phone_enc`、`student_no_enc` 只在后台管理接口解密使用。
