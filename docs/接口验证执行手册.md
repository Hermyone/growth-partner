## 二、认证模块（Auth）— 5 条路由

### 2.1 路由清单

| 方法 | 路径 | 功能 |
| :--- | :--- | :--- |
| POST | /api/v1/auth/login | 统一登录 |
| POST | /api/v1/auth/refresh | Token 刷新 |
| POST | /api/v1/auth/logout | 注销登录 |
| GET | /api/v1/auth/me | 获取当前用户信息 |
| PATCH | /api/v1/auth/password | 修改密码 |

### 2.2 验证维度

#### ① 功能正确性

| 测试场景 | 预期结果 | 测试命令 |
| :--- | :--- | :--- |
| 老师登录 | 返回 access_token 和用户信息 | `curl -X POST http://localhost:8080/api/v1/auth/login -H "Content-Type: application/json" -d '{"username":"teacher001","password":"123456","role":"teacher"}'` |
| 学生登录 | 返回 access_token 和用户信息（含 child_id） | `curl -X POST http://localhost:8080/api/v1/auth/login -H "Content-Type: application/json" -d '{"username":"student001","password":"123456","role":"student"}'` |
| 家长登录 | 返回 access_token 和用户信息 | `curl -X POST http://localhost:8080/api/v1/auth/login -H "Content-Type: application/json" -d '{"username":"parent001","password":"123456","role":"parent"}'` |
| 密码错误 | 返回 401 错误 | `curl -X POST http://localhost:8080/api/v1/auth/login -H "Content-Type: application/json" -d '{"username":"teacher001","password":"wrong","role":"teacher"}'` |
| 角色不匹配 | 返回 401 错误 | `curl -X POST http://localhost:8080/api/v1/auth/login -H "Content-Type: application/json" -d '{"username":"teacher001","password":"123456","role":"student"}'` |

#### ② 安全性

| 测试场景 | 预期结果 | 测试命令 |
| :--- | :--- | :--- |
| 暴力破解防护 | 10秒内5次失败后返回 429 | `for i in {1..6}; do curl -s -X POST http://localhost:8080/api/v1/auth/login -H "Content-Type: application/json" -d '{"username":"teacher001","password":"wrong","role":"teacher"}'; sleep 1; done` |
| Token 伪造 | 服务器拒绝 | 修改 JWT payload 后重新编码测试 |
| 并发登录 | Token 一致性 | 同时发起 100 个登录请求 |
| 重复 logout | 第二次返回 401 | 同一 token 注销两次 |

#### ③ 数据一致性

| 测试场景 | 预期结果 | 测试命令 |
| :--- | :--- | :--- |
| 修改密码后旧 token 失效 | 旧 token 访问失败 | 修改密码后使用旧 token 访问 /auth/me |
| refresh_token 单次使用 | 使用一次后失效 | 使用 refresh_token 刷新后再次使用 |

## 十、测试执行顺序

```bash
# 1. 环境准备
docker compose up -d
sleep 10  # 等待服务就绪

# 2. 健康检查
curl http://localhost:8080/health

# 3. 初始化种子数据
ADMIN_TOKEN=$(curl -s -X POST http://localhost:8080/api/v1/auth/login -H "Content-Type: application/json" -d '{"username":"admin","password":"admin123","role":"admin"}' | jq -r '.data.access_token')
curl -X POST http://localhost:8080/api/v1/admin/partner-templates/seed \
  -H "Authorization: Bearer $ADMIN_TOKEN"

# 4. 按优先级执行 Newman 测试集合
newman run auth_collection.json -e env.json
newman run admin_collection.json -e env.json
newman run teacher_collection.json -e env.json
newman run student_collection.json -e env.json
newman run parent_collection.json -e env.json
newman run battle_collection.json -e env.json

# 5. WebSocket 专项测试
go test ./test/websocket/... -v -timeout 120s

# 6. 压力测试
wrk -t12 -c400 -d30s http://localhost:8080/api/v1/partner-templates
```
